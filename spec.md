# 多元线性回归分析工具 Spec

> 状态：已确认并按本规格完成实施
> 版本：v0.1 需求草案
> 最后更新：2026-07-16

## 1. 产品目标

在现有项目中新增一个纯前端、无需登录的多元线性回归分析工具。用户可以上传或粘贴 CSV 数据，选择一个应变量和一个或多个自变量，一键得到可解释、可导出的完整 OLS（普通最小二乘）回归结果与诊断图表。

工具的核心原则：

- 所有数据解析、计算和绘图都在用户浏览器本地完成。
- 原始数据不上传服务器，不写入远端数据库，也不调用远程分析 API。
- 页面可部署到 GitHub Pages 等纯静态托管环境。
- 面向业务分析和教学场景，默认界面使用中文，保留通用英文统计缩写。
- 结果页面必须同时给出数值、含义和必要警告，不只展示“显著/不显著”。

## 3. 首版用户流程

1. 用户打开工具页，看到简短说明、隐私提示和数据要求。
2. 用户通过以下任一方式导入 CSV：
   - 拖拽或点击选择本地 `.csv` 文件；
   - 将 CSV 文本直接粘贴到输入框；
   - 点击“加载示例数据”体验完整流程。
3. 工具解析数据并展示：文件名、编码/分隔符、行数、列数、字段类型和前若干行预览。
4. 用户选择：
   - 一个应变量 Y；
   - 一个或多个自变量 X；
   - 是否包含截距项，默认开启；
   - 置信水平，默认 95%。
5. 页面即时显示数据可用性摘要和阻止分析的问题。
6. 用户点击“一键运行回归”。
7. 页面生成模型摘要、ANOVA、系数表、诊断结果和图表。
8. 用户可以修改变量后重新分析、下载结果，或清空当前数据。

首版不需要多页面向导；桌面端优先采用左侧配置、右侧结果或上方配置、下方结果的单页结构，移动端改为纵向排列。

## 4. 数据输入与解析

### 4.1 支持范围

- 文件格式：`.csv` 和可按 CSV 解析的纯文本。
- 第一行必须为字段名。
- 支持标准 CSV 引号、引号内换行、转义引号和空字段。
- 自动识别常见分隔符：逗号、制表符、分号。
- 默认支持 UTF-8；检测到非 UTF-8 时可尝试浏览器支持的 GB18030/GBK 解码，并明确显示实际解码结果或失败原因。
- 重复列名必须被识别并阻止分析，不能静默改名。
- 空白列名必须报错并提示用户修正。

### 4.2 字段类型

解析后为每一列识别：

- 数值型；
- 非数值型；
- 全空列；
- 数值型但包含缺失/非法值。

Y 和 X 首版只允许选择数值型字段。日期、文本和类别字段仍可在预览中显示，但不可直接进入模型。

数值识别规则：

- 去除值前后空白；
- 支持普通整数、小数、负数和科学计数法；
- 空字符串、`NA`、`N/A`、`null`、`NULL`、`NaN` 视为缺失值；
- 不自动移除千位逗号、百分号、货币符号或单位，避免错误转换；相应列标记为非数值或包含非法值，并指出示例行。

### 4.3 数据规模

首版建议边界：

- 文件大小软上限：10 MB；
- 数据行数软上限：50,000 行；
- 可选自变量上限：50 个；
- 超过软上限时给出性能警告，可在浏览器能力允许时继续；
- 图表最多绘制 5,000 个代表性点，统计计算仍使用全部有效行，并在图表旁标注抽样展示。

页面不得因大文件长时间无响应。解析和计算期间显示进度/忙碌状态，按钮防止重复触发；若实际实现验证表明主线程计算不够流畅，应使用 Web Worker，但仍保持纯前端。

### 4.4 缺失值与有效样本

模型使用 complete-case/listwise deletion：只要某行在当前选择的 Y 或任一 X 中缺失或非法，该行就不进入模型。

运行前后都必须展示：

- 原始总行数；
- 有效样本数；
- 被剔除行数和占比；
- 每个所选变量的缺失/非法值数量；
- 被剔除的原始行号，允许随诊断明细一起导出。

不自动插值、不以均值填补，也不静默把缺失值变为 0。

## 5. 模型定义与计算要求

### 5.1 模型

首版实现普通最小二乘多元线性回归：

```text
Y = β₀ + β₁X₁ + β₂X₂ + ... + βₖXₖ + ε
```

- 默认包含截距 `β₀`；用户可以关闭截距。
- 不做自动变量选择、逐步回归、正则化或因果推断。
- 不自动标准化原始变量；系数表额外计算标准化系数，便于比较量纲不同的 X。
- 行顺序仅用于 Durbin–Watson 等序列诊断，工具不会把任意横截面数据自动解释为时间序列。

### 5.2 数值算法

- 系数估计应使用数值稳定的 QR、带列主元 QR 或 SVD 方法，不以直接求逆 `(X'X)⁻¹X'Y` 作为主算法。
- 统计分布函数和矩阵运算可以使用固定版本、随项目本地保存的成熟开源前端库。
- 页面运行不得依赖 CDN、远程脚本或联网状态。
- 若设计矩阵秩不足、接近奇异或条件数过高，必须停止或明确降级，并指出可能重复/共线的变量；不得输出看似正常但不可信的系数。
- 所有展示数值内部使用完整精度计算；界面仅做格式化，不用显示值反向参与计算。

### 5.3 分析前校验

以下情况阻止运行并提供可操作提示：

- 未选择 Y 或未选择 X；
- Y 同时出现在 X 中；
- 有效样本数 `n` 不大于待估参数个数；
- Y 没有变异；
- 任一 X 为常数列；
- 出现无穷大或无法解析的数值；
- 设计矩阵不满秩，无法唯一估计系数；
- 关闭截距时，相关统计量的定义与含截距模型不同，页面必须显著提示。

## 6. 统计结果

### 6.1 分析概览

顶部摘要至少包括：

- 应变量名称；
- 自变量列表；
- 原始样本数、有效样本数、剔除样本数；
- 是否包含截距；
- 置信水平；
- 回归方程（变量较多时可折叠/横向滚动）；
- 运行时间；
- 主要警告数量。

### 6.2 模型拟合摘要

至少输出：

- Multiple R；
- R²；
- Adjusted R²；
- 残差标准误（Residual Standard Error）；
- SSE、SSR、SST；
- RMSE；
- MAE；
- 样本数 `n`；
- 自变量数 `k`；
- 模型、残差和总自由度；
- F 统计量及其 p 值；
- AIC 和 BIC（基于带正态误差的 OLS 似然，并在界面说明口径）。

关闭截距时，必须使用无截距模型相应的未中心化统计量，并明确标注 `Uncentered R²`，避免与普通 R² 混淆。

### 6.3 ANOVA 表

至少包含：

| 来源 | 平方和 SS | 自由度 df | 均方 MS | F | p 值 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 回归 | SSR | 模型 df | MSR | F | p |
| 残差 | SSE | 残差 df | MSE |  |  |
| 总计 | SST | 总 df |  |  |  |

### 6.4 系数表

每个截距/自变量至少输出：

- 变量名；
- 非标准化系数 B；
- 标准误 SE；
- 标准化系数 Beta（截距留空）；
- t 统计量；
- 双侧 p 值；
- 置信区间下限与上限；
- VIF；
- Tolerance；
- 显著性标记，并附标记说明。

VIF/Tolerance 只对自变量计算。若统计量因模型设置或数值问题无定义，显示 `N/A` 及原因，不显示误导性的 0。

### 6.5 残差与影响诊断

至少输出整体诊断摘要：

- 残差最小值、Q1、中位数、Q3、最大值；
- Durbin–Watson；
- Jarque–Bera 正态性检验及 p 值；
- Breusch–Pagan 异方差检验及 p 值；
- 最大 VIF；
- 条件数/条件指标；
- 高杠杆点数量；
- 大残差点数量；
- 高 Cook's distance 点数量。

逐行诊断明细至少包含：

- 原始 CSV 行号；
- 实际值；
- 预测值；
- 原始残差；
- 标准化/学生化残差；
- leverage（hat value）；
- Cook's distance；
- 异常标记。

默认标记参考阈值：

- `|studentized residual| > 2`：可能的大残差；
- `leverage > 2p/n`：可能的高杠杆点，其中 `p` 为待估参数数目；
- `Cook's D > 4/n`：可能的高影响点；
- `VIF > 5`：值得关注，`VIF > 10`：严重共线性风险。

这些阈值只用于提示，不自动删除样本，也不自动重新拟合模型。

### 6.6 解释与警告

结果页使用简洁中文解释以下概念：

- R²/Adjusted R² 反映拟合程度，不等于因果关系；
- 系数是在其他所选变量保持不变时的条件关联；
- p 值不是效应大小，也不代表原假设为真的概率；
- 多重共线性会使单个系数不稳定；
- 异方差、非正态、高影响点和序列相关会影响常规推断；
- 工具输出用于分析辅助，不能替代数据背景和研究设计。

警告应根据当前结果动态出现，不使用固定的“模板结论”。

## 7. 可视化

首版至少包含以下交互或响应式图表：

1. 实际值 vs 预测值散点图，含 45° 参考线；
2. 残差 vs 拟合值散点图，含 `y = 0` 参考线和平滑趋势提示；
3. 残差直方图，叠加基于样本均值/标准差的正态参考曲线；
4. 残差正态 Q–Q 图，含参考线；
5. Scale–Location 图；
6. Leverage vs 学生化残差图，以点大小或颜色表达 Cook's distance。

图表要求：

- 轴标题必须包含实际变量名或统计量名；
- 悬停显示原始行号和关键数值；
- 高影响点使用视觉标记；
- 不只依靠颜色传达异常状态；
- 在手机屏幕上可阅读，不发生整页横向溢出；
- 大样本抽样只影响绘图，不影响统计结果，并明确标注抽样规则。

首版不要求三维回归面、每个 X 的偏回归图或自动生成文字研究报告。

## 8. 结果组织与导出

结果区建议按以下标签或章节组织：

1. 概览；
2. 模型摘要；
3. ANOVA；
4. 系数；
5. 诊断；
6. 图表；
7. 数据明细。

至少提供：

- 下载模型摘要 CSV；
- 下载系数表 CSV；
- 下载逐行预测与诊断 CSV；
- 下载包含配置、统计量和警告的 JSON；
- 浏览器打印样式，允许用户“打印/另存为 PDF”。

导出内容必须包含分析时间、Y、X、截距设置、置信水平、原始/有效样本数和缺失值处理说明，保证结果可追溯。

首版不导出用户上传的完整原始数据副本，不自动保存分析历史，不使用浏览器云同步。

## 9. 页面与交互要求

### 9.1 视觉结构

- 延续现有汇总页的简洁研究工具风格，但工具页可以使用更适合分析工作台的卡片、表格和标签页。
- 首屏清楚展示“数据仅在本地浏览器处理”。
- 主操作按钮统一为“一键运行回归”。
- 表格列较多时容器内横向滚动，固定变量名列和表头（浏览器允许时）。
- 统计结果使用合适的小数/科学计数格式；p 值很小时显示如 `< 0.001`，导出保留更高精度。

### 9.2 状态处理

必须设计以下状态：

- 初始空状态；
- 拖拽悬停状态；
- 解析中/计算中；
- 解析成功；
- 可修复的字段警告；
- 阻止运行的错误；
- 无有效样本；
- 计算成功；
- 清空并重新开始。

错误信息应说明错误位置和修复方式，例如指出列名、原始行号或冲突变量，而不是只显示“分析失败”。

### 9.3 可访问性

- 所有控件可以使用键盘操作；
- 文件拖拽区同时提供标准文件选择按钮；
- 表单控件具备可见标签；
- 图表有标题和文字摘要；
- 颜色对比度满足常规可读性；
- `prefers-reduced-motion` 下减少非必要动画。

## 10. 技术约束

- 纯 HTML/CSS/JavaScript 静态应用，不需要后端、数据库或账号系统。
- 无需 Node.js 或打包命令即可从静态服务器运行。
- 不使用 `eval`，不执行用户 CSV 中的任何脚本或 HTML。
- CSV 预览和字段名必须以文本方式安全渲染，防止注入。
- 所有第三方前端依赖必须固定版本、本地保存，并在 README 中注明名称、版本和许可证。
- 不发起分析、遥测、广告或错误上报网络请求。
- 目标浏览器：当前主流桌面版 Chrome、Edge、Safari、Firefox；移动端提供可用布局。
- 直接双击 `index.html` 的支持作为加分项；正式验收以 GitHub Pages/本地静态服务器环境为准。

## 11. 非目标（首版不做）

- 类别变量自动哑编码；
- 交互项、多项式项、Box–Cox 等自动特征变换；
- 加权最小二乘、稳健回归、岭回归、Lasso、Logistic 回归；
- 面板、时间序列或混合效应模型；
- 自动变量筛选、逐步回归或“最优模型”推荐；
- 自动删除异常点；
- 缺失值自动填补；
- 因果推断或因果措辞；
- 服务端保存、协作分享链接或用户账户；
- 直接读取 Excel/SPSS/Stata 文件。

上述能力可作为后续版本选项，但不应延迟首版交付。

## 12. 测试与验收标准

### 12.1 统计正确性

准备至少四类固定测试数据：

1. 小型、无缺失、满秩数据；
2. 含缺失和非法值的数据；
3. 完全共线/近共线数据；
4. 含明显异常点和异方差的数据。

将有效模型的结果与 Python `statsmodels` 或 R `lm()` 对照，至少验证：

- 系数、标准误、t 值、p 值、置信区间；
- R²、Adjusted R²、F 和 F 检验 p 值；
- ANOVA 平方和与自由度；
- 预测值、残差、leverage 和 Cook's distance；
- VIF、Durbin–Watson、Jarque–Bera、Breusch–Pagan；
- 含截距和不含截距两种模式。

建议数值容差：常规良态数据绝对/相对误差 `1e-6`；不同统计库定义可能存在差异的指标须在测试说明中记录口径，不能以放宽容差掩盖公式错误。

### 12.2 功能验收

- 上传、拖拽、粘贴和示例数据四条路径均可完成分析；
- 变量选择不会允许 Y/X 重复；
- 改变 Y、X、截距或置信水平后必须重新计算，不展示过期结果；
- 缺失行计数、原始行号映射和导出文件一致；
- 诊断阈值标记与逐行明细一致；
- 所有下载文件可以打开，字段完整且中文不乱码；
- 刷新页面后不残留之前上传的数据；
- 离线环境下核心分析、图表和导出仍可用；
- 汇总页的新工具入口能正确打开工具页；
- GitHub Pages 路径下资源引用正常，不使用只在本机有效的绝对路径。

### 12.3 体验验收

- 1366px 桌面宽度和 390px 手机宽度下无关键内容遮挡；
- 键盘可以完成数据导入后的核心配置和运行；
- 典型 10,000 行、10 个 X 的数据在目标桌面浏览器中应保持可操作，并记录实测耗时；
- 页面不产生意外网络请求或 JavaScript 控制台错误。

如用户直接回复“确认”，即视为同意以下首版默认值：

1. 首版仅允许数值型 Y 和 X，不自动处理类别变量。
